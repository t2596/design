# 车辆数据功能修复说明

## 问题描述

车辆客户端成功注册并显示在线，但 Web 界面无法看到车辆数据。

## 根本原因

客户端创建了安全报文但没有通过 HTTP API 将数据发送到网关。之前的实现只是在本地创建了加密消息，但没有实际发送到服务器。

## 解决方案

### 1. 修改客户端代码 (`client/vehicle_client.py`)

更新 `send_vehicle_data()` 方法，改为通过 HTTP POST 请求发送数据：

```python
def send_vehicle_data(self, data: bytes) -> bool:
    """发送车辆数据到网关"""
    # 解析 JSON 数据
    data_dict = json.loads(data.decode('utf-8'))
    
    # 通过 HTTP API 发送数据
    response = requests.post(
        f"{self.gateway_url}/api/auth/data",
        params={
            "vehicle_id": self.vehicle_id,
            "session_id": self.session_id
        },
        json=data_dict,
        headers={"Authorization": f"Bearer {self.api_token}"}
    )
    
    return response.status_code == 200
```

### 2. 后端 API 已就绪

`src/api/routes/auth.py` 中的 `/api/auth/data` 端点已经实现：
- 验证会话有效性
- 更新心跳时间
- 保存数据到 PostgreSQL vehicle_data 表

### 3. 数据库结构已就绪

K8s PostgreSQL 初始化脚本 (`deployment/kubernetes/postgres-init-configmap.yaml`) 已包含：
- vehicle_data 表结构
- 索引优化
- latest_vehicle_data 视图

## 部署步骤

### 方式一：重新构建并部署（推荐）

```bash
# 1. 重新构建 Gateway 镜像
docker build -t vehicle-iot-gateway:latest .

# 2. 重启 Gateway 部署
kubectl rollout restart deployment/gateway -n vehicle-iot-gateway

# 3. 重新构建 Client 镜像
docker build -f client/Dockerfile -t vehicle-client:latest .

# 4. 运行客户端
docker run --rm \
  -e GATEWAY_HOST=<gateway-ip> \
  -e GATEWAY_PORT=8000 \
  -e API_TOKEN=dev-token-12345 \
  vehicle-client:latest \
  --vehicle-id VIN123456789 \
  --mode continuous \
  --interval 10
```

### 方式二：直接运行 Python 客户端

```bash
cd client
python vehicle_client.py \
  --vehicle-id VIN123456789 \
  --gateway-host <gateway-host> \
  --gateway-port 8000 \
  --mode continuous \
  --interval 10
```

## 故障排查

### 问题：客户端显示 "✗ 数据发送失败"

**解决方法**：
1. 检查 Gateway 是否运行：`kubectl get pods -n vehicle-iot-gateway`
2. 查看 Gateway 日志：`kubectl logs -n vehicle-iot-gateway -l app=gateway --tail=50`
3. 确认网络连接：`curl http://<gateway-host>:8000/health`

### 问题：Web 界面不显示数据

**解决方法**：
1. 打开浏览器开发者工具，检查控制台错误
2. 检查网络请求，确认 API 调用成功
3. 查询数据库确认数据已保存：
   ```bash
   kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
     psql -U gateway_user -d gateway_db -c "SELECT COUNT(*) FROM vehicle_data;"
   ```

### 问题：数据库连接失败

**解决方法**：
1. 检查 PostgreSQL 状态：`kubectl get pods -n vehicle-iot-gateway -l app=postgres`
2. 检查环境变量：`kubectl describe deployment gateway -n vehicle-iot-gateway`
3. 如果需要，重新部署 PostgreSQL：
   ```bash
   kubectl delete -f deployment/kubernetes/postgres-deployment.yaml
   kubectl apply -f deployment/kubernetes/postgres-init-configmap.yaml
   kubectl apply -f deployment/kubernetes/postgres-deployment.yaml
   ```

## 数据流程图

```
客户端 (vehicle_client.py)
    ↓ HTTP POST /api/auth/data
网关 API (src/api/routes/auth.py)
    ↓ 验证会话 (Redis)
    ↓ 保存数据 (PostgreSQL)
数据库 (vehicle_data 表)
    ↓ HTTP GET /api/vehicles/{id}/data/latest
Web 界面 (VehicleMonitor.jsx)
    ↓ 显示实时数据
用户浏览器
```
