# 车辆在线状态管理优化

## 问题描述

客户端关闭后，Web 界面仍然显示车辆在线。这是因为 Redis 中的会话数据有 30 分钟的过期时间，在客户端关闭后会话仍然存在。

## 解决方案

实现了两层防护机制：

### 1. 客户端优雅退出（主动注销）

**修改文件**: `client/vehicle_client.py`

#### 新增功能

1. **添加 `unregister()` 方法**
   - 调用 `/api/auth/unregister` 端点
   - 从 Redis 中删除会话数据
   - 清理车辆到会话的映射

2. **在退出时自动注销**
   - 单次模式：完成后自动注销
   - 连续模式：捕获 Ctrl+C 信号，优雅退出并注销
   - 异常退出：也会尝试注销

#### 代码示例

```python
def unregister(self) -> bool:
    """注销车辆"""
    response = requests.post(
        f"{self.gateway_url}/api/auth/unregister",
        params={
            "vehicle_id": self.vehicle_id,
            "session_id": self.session_id
        },
        headers={"Authorization": f"Bearer {self.api_token}"}
    )
    return response.status_code == 200

def run_continuous_mode(self, interval: int = 5, max_iterations: int = 0):
    try:
        # ... 数据传输循环 ...
    except KeyboardInterrupt:
        print("用户中断，正在注销...")
        self.unregister()  # 优雅退出
    except Exception as e:
        print(f"运行异常: {e}")
        self.unregister()  # 异常退出也注销
```

### 2. 后端会话超时检测（被动清理）

**修改文件**: `src/api/routes/vehicles.py`

#### 超时策略

- **超时时间**: 5 分钟（300 秒）
- **检测时机**: 查询在线车辆列表或车辆状态时
- **处理方式**: 自动删除超时会话

#### 实现逻辑

1. **在 `get_online_vehicles()` 中**
   ```python
   current_time = datetime.utcnow()
   timeout_seconds = 300  # 5 分钟
   
   for session in sessions:
       last_activity = session['last_activity_time']
       time_diff = (current_time - last_activity).total_seconds()
       
       if time_diff <= timeout_seconds:
           # 在线车辆
           vehicles.append(session)
       else:
           # 超时，删除会话
           redis_conn.delete(session_key)
           redis_conn.delete(vehicle_key)
   ```

2. **在 `get_vehicle_status()` 中**
   - 检查单个车辆的最后活动时间
   - 超过 5 分钟返回 "offline" 状态
   - 同时删除过期会话

## 工作流程

### 正常退出流程

```
1. 用户按 Ctrl+C 或程序完成
   ↓
2. 客户端捕获退出信号
   ↓
3. 调用 unregister() 方法
   ↓
4. 发送 POST /api/auth/unregister
   ↓
5. 后端删除 Redis 会话数据
   ↓
6. Web 界面立即不再显示该车辆
```

### 异常退出流程（客户端崩溃）

```
1. 客户端异常终止（无法注销）
   ↓
2. Redis 会话仍然存在
   ↓
3. 5 分钟内无新的心跳
   ↓
4. Web 界面查询在线车辆
   ↓
5. 后端检测到会话超时
   ↓
6. 自动删除过期会话
   ↓
7. Web 界面不再显示该车辆
```

## 配置参数

### 会话超时时间

可以根据需要调整超时时间：

**当前设置**: 5 分钟（300 秒）

**修改位置**: `src/api/routes/vehicles.py`

```python
timeout_seconds = 300  # 修改这个值
```

**建议值**:
- 开发测试: 60-120 秒（1-2 分钟）
- 生产环境: 300-600 秒（5-10 分钟）

### Redis 会话过期时间

**当前设置**: 30 分钟（1800 秒）

**修改位置**: `src/api/routes/auth.py`

```python
redis_conn.set(session_key, session_data, ex=1800)  # 修改 ex 参数
```

**注意**: Redis 过期时间应该大于超时检测时间，作为最后的兜底机制。

## 总结

通过双重机制确保车辆在线状态的准确性：

1. **主动注销**: 客户端正常退出时立即清理会话
2. **被动清理**: 后端定期检测并清理超时会话

这样即使客户端异常崩溃无法注销，最多 5 分钟后也会自动从在线列表中移除。
