# 手动修复数据库

## 步骤 1: 检查 PostgreSQL Pod 状态

```bash
kubectl get pods -n vehicle-iot-gateway -l app=postgres
```

## 步骤 2: 查看现有数据库

```bash
kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  psql -U postgres -c "\l"
```

## 步骤 3: 手动创建数据库和用户

```bash
# 创建数据库
kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  psql -U postgres -c "CREATE DATABASE gateway_db;"

# 创建用户（如果不存在）
kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  psql -U postgres -c "CREATE USER gateway_user WITH PASSWORD 'gateway_password';"

# 授予权限
kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  psql -U postgres -c "GRANT ALL PRIVILEGES ON DATABASE gateway_db TO gateway_user;"
```

## 步骤 4: 创建表结构

```bash
# 方法 1: 使用初始化脚本
kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  psql -U gateway_user -d gateway_db < /docker-entrypoint-initdb.d/01-schema.sql

# 方法 2: 手动执行 SQL（如果方法 1 失败）
# 复制 postgres-init-configmap.yaml 中的 SQL 内容并执行
```

## 步骤 5: 验证表是否创建

```bash
kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  psql -U gateway_user -d gateway_db -c "\dt"
```

应该看到：
- certificates
- certificate_revocation_list
- audit_logs
- vehicle_data

## 步骤 6: 测试插入数据

```bash
kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  psql -U gateway_user -d gateway_db -c \
  "INSERT INTO vehicle_data (vehicle_id, timestamp, state, motion_speed) VALUES ('TEST', NOW(), '测试', 100);"

# 查询测试数据
kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  psql -U gateway_user -d gateway_db -c \
  "SELECT * FROM vehicle_data WHERE vehicle_id='TEST';"
```

## 如果上述方法都失败

### 完全重新部署 PostgreSQL

```bash
# 1. 删除 PostgreSQL 部署
kubectl delete -f deployment/kubernetes/postgres-deployment.yaml

# 2. 删除 PVC（如果使用了持久卷）
kubectl get pvc -n vehicle-iot-gateway
kubectl delete pvc postgres-pvc -n vehicle-iot-gateway

# 3. 重新应用 ConfigMap
kubectl apply -f deployment/kubernetes/postgres-init-configmap.yaml

# 4. 重新部署 PostgreSQL
kubectl apply -f deployment/kubernetes/postgres-deployment.yaml

# 5. 等待 Pod 就绪
kubectl wait --for=condition=ready pod -l app=postgres -n vehicle-iot-gateway --timeout=120s

# 6. 等待额外 30 秒让初始化脚本执行
sleep 30

# 7. 验证数据库
kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  psql -U postgres -c "\l" | grep gateway_db
```

## 检查初始化日志

```bash
# 查看 PostgreSQL 启动日志
kubectl logs -n vehicle-iot-gateway -l app=postgres --tail=100

# 查找初始化相关的日志
kubectl logs -n vehicle-iot-gateway -l app=postgres | grep -i "init\|database\|gateway_db"
```

## 常见问题

### 问题 1: 初始化脚本没有执行

**原因**: PVC 中已有数据，PostgreSQL 跳过初始化

**解决**: 删除 PVC 重新部署

### 问题 2: 权限问题

**症状**: "permission denied" 或 "role does not exist"

**解决**:
```bash
kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  psql -U postgres -c "ALTER USER gateway_user WITH SUPERUSER;"
```

### 问题 3: ConfigMap 未挂载

**检查**:
```bash
kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  ls -la /docker-entrypoint-initdb.d/
```

应该看到 `01-schema.sql` 和 `02-init-data.sql`

如果没有，检查 postgres-deployment.yaml 中的 volumeMounts 配置。
