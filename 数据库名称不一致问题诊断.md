## 数据库初始化问题

### PostgreSQL 初始化脚本位置

文件：`deployment/kubernetes/postgres-init-configmap.yaml`

```yaml
data:
  01-schema.sql: |
    -- 创建 certificates 表
    -- 创建 audit_logs 表
    -- 创建 vehicle_data 表
    -- 创建视图 latest_vehicle_data
```

### 初始化脚本挂载

文件：`deployment/kubernetes/postgres-deployment.yaml`

```yaml
volumeMounts:
- name: init-scripts
  mountPath: /docker-entrypoint-initdb.d  # ← PostgreSQL 自动执行此目录下的 .sql 文件

volumes:
- name: init-scripts
  configMap:
    name: postgres-init-scripts
```

### 初始化条件

PostgreSQL 官方镜像的初始化逻辑：
1. **只在数据目录为空时执行** `/docker-entrypoint-initdb.d` 中的脚本
2. 如果数据目录已存在（即使数据库不完整），初始化脚本**不会执行**
3. 使用 `emptyDir` 卷意味着每次 Pod 重启数据都会丢失，但会重新初始化

## 修复方案

### 方案 1: 完全重新部署 PostgreSQL（推荐）

```bash
# 1. 删除 PostgreSQL 部署
kubectl delete deployment postgres -n vehicle-iot-gateway

# 2. 等待 Pod 完全删除
kubectl wait --for=delete pod -l app=postgres -n vehicle-iot-gateway --timeout=60s

# 3. 重新应用 ConfigMap（确保最新）
kubectl apply -f deployment/kubernetes/postgres-init-configmap.yaml

# 4. 重新部署 PostgreSQL
kubectl apply -f deployment/kubernetes/postgres-deployment.yaml

# 5. 等待 Pod 就绪
kubectl wait --for=condition=ready pod -l app=postgres -n vehicle-iot-gateway --timeout=120s

# 6. 检查初始化日志
kubectl logs -n vehicle-iot-gateway -l app=postgres --tail=100

# 7. 验证数据库
kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  psql -U postgres -c "\l" | grep vehicle_iot_gateway

# 8. 验证表
kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  psql -U gateway_user -d vehicle_iot_gateway -c "\dt"
```

### 方案 2: 手动创建数据库和表

如果方案 1 失败，手动执行：

```bash
# 1. 创建数据库（如果不存在）
kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  psql -U postgres -c "CREATE DATABASE vehicle_iot_gateway;"

# 2. 授予权限
kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  psql -U postgres -c "GRANT ALL PRIVILEGES ON DATABASE vehicle_iot_gateway TO gateway_user;"

# 3. 获取初始化脚本内容
kubectl get configmap postgres-init-scripts -n vehicle-iot-gateway -o jsonpath='{.data.01-schema\.sql}' > /tmp/schema.sql

# 4. 复制到 Pod
kubectl cp /tmp/schema.sql vehicle-iot-gateway/$(kubectl get pod -n vehicle-iot-gateway -l app=postgres -o jsonpath='{.items[0].metadata.name}'):/tmp/schema.sql

# 5. 执行初始化脚本
kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  psql -U gateway_user -d vehicle_iot_gateway -f /tmp/schema.sql
```

## 常见问题

### Q1: 为什么数据库名称不一致？

A: 这是配置错误。应该统一使用 `vehicle_iot_gateway`（ConfigMap 中配置的名称）。

### Q2: 初始化脚本为什么没有执行？

A: PostgreSQL 只在数据目录为空时执行初始化脚本。如果之前部署过，数据目录可能已存在。

### Q3: 如何确保初始化脚本执行？

A: 完全删除 PostgreSQL Deployment，等待 Pod 删除后重新部署。使用 `emptyDir` 卷会在 Pod 删除时清空数据。

### Q4: 客户端显示成功但数据库没有数据？

A: 检查：
1. Gateway 日志是否有错误
2. 数据库连接是否正确
3. 表是否存在
4. 权限是否正确
