# 车辆数据流问题 - 完整诊断报告

## 执行摘要

经过全面检查，发现了导致"客户端传的数据在 Web 端看不到"的**根本原因**：

### 🔴 核心问题：数据库名称配置不一致

**问题描述**：
- **ConfigMap 配置**：`POSTGRES_DB=vehicle_iot_gateway`
- **文档和脚本使用**：`gateway_db`
- **结果**：数据库根本不存在，导致所有数据写入失败

**问题**：

- PostgreSQL 初始化脚本可能没有执行

**错误信息**：

**状态**：✗ 数据库不存在，数据写入失败

### 4. PostgreSQL 初始化（✗ 可能未执行）

**文件**：`deployment/kubernetes/postgres-init-configmap.yaml`

**初始化脚本**：
- `01-schema.sql`：创建表结构（certificates, audit_logs, vehicle_data）
- `02-init-data.sql`：插入初始数据

**挂载配置**：
```yaml
volumeMounts:
- name: init-scripts
  mountPath: /docker-entrypoint-initdb.d  # PostgreSQL 自动执行此目录的脚本

volumes:
- name: init-scripts
  configMap:
    name: postgres-init-scripts
```

**PostgreSQL 初始化规则**：
1. **只在数据目录为空时执行**初始化脚本
2. 如果数据目录已存在（即使不完整），脚本**不会执行**
3. 使用 `emptyDir` 卷，Pod 删除时数据会清空

**可能的问题**：
- 之前部署过，数据目录已存在但不完整
- 初始化脚本没有执行
- 数据库创建失败但 Pod 仍在运行

**状态**：✗ 初始化可能失败

## 问题根源总结

### 主要问题

1. **数据库名称不一致**
   - ConfigMap: `vehicle_iot_gateway`
   - 文档/脚本: `gateway_db`
   - 结果：混乱和错误

2. **数据库初始化失败**
   - 初始化脚本可能没有执行
   - 数据库 `vehicle_iot_gateway` 不存在
   - 表结构不存在

3. **代码已修复但数据库问题未解决**
   - `execute_query` → `execute_update` 已修复
   - 但数据库不存在，修复无效

### 次要问题

1. **错误信息不明确**
   - "no results to fetch" 不能准确反映问题
   - 应该是 "database does not exist"

2. **文档不一致**
   - 所有诊断文档使用错误的数据库名
   - 导致用户无法正确诊断问题

## 常见问题

### Q1: 为什么客户端显示成功但数据库没有数据？

A: 因为 Gateway 返回了 200 状态码，但实际上数据库写入失败了。这是因为：
- 数据库不存在
- 异常被捕获但没有正确处理

### Q2: 如何确保初始化脚本执行？

A: 完全删除 PostgreSQL Deployment，等待 Pod 删除后重新部署。使用 `emptyDir` 卷会在 Pod 删除时清空数据目录。

### Q3: 数据库名称应该用哪个？

A: 统一使用 `vehicle_iot_gateway`（ConfigMap 中配置的名称）。

### Q4: 如何实时监控数据写入？

A: 使用以下命令：
```bash
watch -n 2 'kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  psql -U gateway_user -d vehicle_iot_gateway -tAc \
  "SELECT COUNT(*) FROM vehicle_data;"'
```
