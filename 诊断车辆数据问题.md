# 车辆数据发送失败诊断指南

## 错误信息

```
✗ 数据发送失败: 500 - {"detail":"接收车辆数据失败: no results to fetch"}
```

## 问题原因

**根本原因**: `src/api/routes/auth.py` 中的 `receive_vehicle_data()` 函数使用了错误的数据库方法。

- **错误**: 使用 `execute_query()` 执行 INSERT 语句
- **正确**: 应该使用 `execute_update()` 执行 INSERT 语句

`execute_query()` 会尝试 `fetchall()` 获取结果集，但 INSERT 语句不返回结果集，导致 "no results to fetch" 错误。

## 已修复

已将 `src/api/routes/auth.py` 第 169 行从：
```python
db_conn.execute_query(query, params)
```

改为：
```python
db_conn.execute_update(query, params)
```

## 其他可能的问题

### 问题 1: 数据库表不存在

**症状**: 错误信息包含 "relation \"vehicle_data\" does not exist"

**检查**:
```bash
kubectl exec -it deployment/postgres -n vehicle-iot-gateway -- \
  psql -U gateway_user -d gateway_db -c "\dt vehicle_data"
```

**解决**: 重新初始化数据库
```bash
kubectl delete -f deployment/kubernetes/postgres-deployment.yaml
kubectl apply -f deployment/kubernetes/postgres-init-configmap.yaml
kubectl apply -f deployment/kubernetes/postgres-deployment.yaml
```

### 问题 2: 数据库连接失败

**症状**: 错误信息包含 "could not connect to server"

**检查**:
```bash
# 检查 PostgreSQL 是否运行
kubectl get pods -n vehicle-iot-gateway -l app=postgres

# 检查 Gateway 环境变量
kubectl describe deployment gateway -n vehicle-iot-gateway | grep -A 10 "Environment:"
```

**解决**: 确保 PostgreSQL 正在运行，环境变量配置正确

### 问题 3: 会话不存在

**症状**: 错误信息包含 "会话不存在或已过期"

**检查**:
```bash
# 检查 Redis 是否运行
kubectl get pods -n vehicle-iot-gateway -l app=redis

# 检查会话是否存在
kubectl exec -it deployment/redis -n vehicle-iot-gateway -- \
  redis-cli KEYS "session:*"
```

**解决**: 确保客户端先注册，然后再发送数据
