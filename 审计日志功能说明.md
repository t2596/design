# 审计日志功能说明

## 当前状态

审计日志功能**已实现但未完全集成**。

### 已实现的功能

1. **审计日志记录器** (`src/audit_logger.py`)
   - ✓ 记录认证事件
   - ✓ 记录数据传输事件
   - ✓ 记录证书操作事件
   - ✓ 查询审计日志（支持多种过滤条件）
   - ✓ 导出审计报告（JSON/CSV 格式）

2. **审计日志 API** (`src/api/routes/audit.py`)
   - ✓ GET `/api/audit/logs` - 查询审计日志
   - ✓ GET `/api/audit/export` - 导出审计报告

3. **数据库表结构**
   - ✓ `audit_logs` 表已创建
   - ✓ 包含必要的索引

### 集成情况

**已集成**：
- ✓ 证书颁发操作（`certificate_manager.py` 中的 `issue_certificate()`）
- ✓ 证书撤销操作（`certificate_manager.py` 中的 `revoke_certificate()`）

**未集成**（需要添加）：
- ✗ 车辆注册/注销操作
- ✗ 车辆数据接收操作
- ✗ 车辆认证操作

**文件**: `src/api/routes/auth.py`

## 事件类型说明

当前支持的事件类型（`src/models/enums.py`）：

```python
class EventType(Enum):
    AUTHENTICATION_SUCCESS = "AUTHENTICATION_SUCCESS"
    AUTHENTICATION_FAILURE = "AUTHENTICATION_FAILURE"
    DATA_ENCRYPTED = "DATA_ENCRYPTED"
    DATA_DECRYPTED = "DATA_DECRYPTED"
    CERTIFICATE_ISSUED = "CERTIFICATE_ISSUED"
    CERTIFICATE_REVOKED = "CERTIFICATE_REVOKED"
```

如果需要更多事件类型，可以在 `EventType` 枚举中添加。

## 审计日志查询示例

### 查询最近 24 小时的所有日志

```python
from datetime import datetime, timedelta
from src.audit_logger import AuditLogger
from config.database import PostgreSQLConfig
from src.db.postgres import PostgreSQLConnection

db_conn = PostgreSQLConnection(PostgreSQLConfig.from_env())
audit_logger = AuditLogger(db_conn)

end_time = datetime.utcnow()
start_time = end_time - timedelta(hours=24)

logs = audit_logger.query_audit_logs(
    start_time=start_time,
    end_time=end_time
)

for log in logs:
    print(f"[{log.timestamp}] {log.event_type.value}: {log.vehicle_id} - {log.details}")

db_conn.close()
```

### 查询特定车辆的所有操作

```python
logs = audit_logger.query_audit_logs(
    vehicle_id="VIN123456789"
)
```

### 查询所有失败的认证尝试

```python
from src.models.enums import EventType

logs = audit_logger.query_audit_logs(
    event_type=EventType.AUTHENTICATION_FAILURE,
    operation_result=False
)
```

示例 API 客户端：

```javascript
// web/src/api/audit.js
import apiClient from './client';

export const getAuditLogs = async (params = {}) => {
  const response = await apiClient.get('/api/audit/logs', { params });
  return response.data;
};

export const exportAuditReport = async (startTime, endTime, format = 'json') => {
  const response = await apiClient.get('/api/audit/export', {
    params: { start_time: startTime, end_time: endTime, format },
    responseType: 'blob'
  });
  return response.data;
};
```

## 性能考虑

### 索引优化

数据库已创建以下索引：

```sql
CREATE INDEX idx_audit_logs_timestamp ON audit_logs(timestamp);
CREATE INDEX idx_audit_logs_vehicle_id ON audit_logs(vehicle_id);
CREATE INDEX idx_audit_logs_event_type ON audit_logs(event_type);
```

### 数据清理

建议定期清理旧的审计日志：

```sql
-- 删除 90 天前的日志
DELETE FROM audit_logs WHERE timestamp < NOW() - INTERVAL '90 days';
```

可以创建定时任务（cron job）自动执行清理。

## 故障排查

### 问题 1: 查询不到审计日志

**检查步骤**：
1. 确认数据库表存在：`\dt audit_logs`
2. 检查表中是否有数据：`SELECT COUNT(*) FROM audit_logs;`
3. 检查 API 是否正常：`curl http://localhost:8000/api/audit/logs`

### 问题 2: 证书操作没有记录日志

**原因**: 证书操作已集成审计日志，应该会自动记录。

**检查**:
```sql
SELECT * FROM audit_logs 
WHERE event_type IN ('CERTIFICATE_ISSUED', 'CERTIFICATE_REVOKED') 
ORDER BY timestamp DESC 
LIMIT 10;
```

### 问题 3: 车辆操作没有记录日志

**原因**: 车辆注册/注销/数据接收操作尚未集成审计日志。

**解决**: 按照上面"完善审计日志集成"部分的说明添加审计日志记录。
